# Relay Capability Matrix

> **How to read this table.**
> Each row is a deliverable the orchestrator can route a brief to.
> "Live-proven?" means the builder has been exercised against the REAL running
> substrate (not just source-pinned or dry-checked) and the prove-mode gate exits 0.
> Flag-gated builders are live-proven only when the corresponding env flag is set.

## Forced Spine (every deliverable, always)

Every deliverable — regardless of stack or substrate — runs the forced spine before
any build steps. This is the non-negotiable quality floor the orchestrator enforces:

| Seq | Department | Verify gate | What it does |
|---|---|---|---|
| 1 | `strategy` | `min:280` | Audience & positioning — understand the client |
| 2 | `research` | `min:280` | Market & competitor research |
| 3 | `branding` | `wcag` | Brand system: palette + tokens, WCAG-verified contrast |
| 4 | `design` | `min:280` | Design guidelines document |
| … | *(branches)* | *(per-deliverable)* | Dynamic branches injected here |
| last | `qa` | `site_consistent` | Acceptance: 1 nav · 1 logo · 1 palette on every page |

## Routing Logic (deterministic floor)

The orchestrator uses a pure-regex scorer (`detectDeliverable`) to pick the deliverable.
No network call required. The LLM can only **upgrade** from the `directus_site` floor;
it can never downgrade a scored result.

| Brief signal pattern | Scores for |
|---|---|
| `landing page`, `one-page`, `squeeze page`, `page de vente`, `pagina di atterraggio` | `landing_page` |
| `brand identity`, `logo design`, `visual identity`, `charte graphique`, `identità del marchio` | `brand_identity` |
| `blog`, `news`, `magazine`, `editorial`, `multi-author`, `actualités`, `rivista`, `notizie` | `wp_site` |
| `shop`, `store`, `e-commerce`, `checkout`, `cart`, `boutique en ligne`, `vendre`, `negozio`, `vendere` | `wp_woocommerce` |
| `app`, `platform`, `SaaS`, `dashboard`, `booking`, `tracker`, `marketplace`, `plateforme`, `piattaforma` | `fullstack_app` |
| `email campaign`, `newsletter blast`, `drip campaign`, `ad creative`, `campagne email` | `campaign` |
| `simple site`, `brochure`, `portfolio`, `business card` (or no clear signal) | `directus_site` |

## Gate Coverage

| Gate suite | What it proves |
|---|---|
| `orchestrator-check.ts` | Low-level primitives: detectDeliverable, composeChain, detectNeeds, chainReason, multilingual classification, LLM-cannot-downgrade invariant |
| `orchestrated-e2e-check.ts` (T39) | Cross-cutting: every deliverable routed correctly from a representative brief (LLM offline/stubbed); spine order; QA-last; branch presence; chainReason; task shape; determinism |
| `wp-check.ts` | WordPress builder: source pins + RELAY_WP=1 live-prove against relay-wp container |
| `ecom-check.ts` | WooCommerce: product catalog sync, cart/checkout page guarantee, schema present |
| `app-check.ts` | Fullstack app: schema DDL apply, RELAY_APP_API prove mode |
| `appapi:check` | App REST API: list + get-by-id round-trip against Postgres |
| `prestashop-check.ts` | PrestaShop: source pins + RELAY_PRESTA=1 live-prove (optional substrate) |
| `cms/check.ts` | CMS registry: resolveBuilder, directus path, feature flags |
| `chain-check.ts` | Live chain page render, privacy/leak gates |
| `board-check.ts` | Board JS: capability tags, chainReason banner, deliverable filter |
